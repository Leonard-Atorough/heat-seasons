# Heat Seasons - Planning Directory

This directory contains all planning, design, and architecture documentation for the Heat Seasons leaderboard application.

## Directory Structure

- **features/** - Feature specifications and requirements
- **architecture/** - System design and technical architecture
- **testing/** - Test plans and strategies
- **design/** - UI/UX design sessions and mockups

## Project Overview

Heat Seasons is a leaderboard application for tracking games of Heat (the boardgame) in a legacy game format.

### Core Requirements

- Track multiple seasons of races
- Minimum 4 races per season
- Support up to 9 racers per race
- Progressive points system (first place = highest points)
- Legacy format tracking across seasons
